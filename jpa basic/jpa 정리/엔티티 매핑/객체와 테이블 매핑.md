# jpa에서 알아야할것 두가지

1. 영속성컨텍스트등의 동작 매커니즘
2. 엔티티를 매핑하는방법

# 객체와 테이블 매핑

- `@Entity`를 클래스레벨에 붙이면 해당 클래스의객체는 JPA가 관리한다. `@Entity`를 붙여야 jpa가 관리한다! 이를 엔티티라한다.
- JPA를 사용하는데 `@Entity`를 클래스레벨에 붙이지않으면 안된다.

## 주의할점

1. 기본 생성자는 필수이다.(jpa 스펙이다.)
2. final키워드는 클래스에도 멤버변수에도 붙이면안됨!

# @Entity

- jpa에서 사용할 엔티티 이름을 지정한다. default로 본인 클래스이름임
- 왠만하면 기본값유지하자. (다른 패키지에 엔티티로 관리할 다른 클래스의이름이 같은경우엔 name으로 값 지정)

# @Table

- name의 속성으로 매핑할 엔티티를 지정할수있다.

## @Entity의 name속성 vs @Table의 name속성

- 전자 : JPA에서 사용할 엔티티의 이름을 지정
- 후자 : 매핑할 테이블의 이름을 지정

> jpa에서 엔티티: 테이블과 매핑되는 클래스
